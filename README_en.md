# Archaeological Map of the Czech Republic (AMCR)

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.8363642.svg)](https://zenodo.org/doi/10.5281/zenodo.8363642) [![SWH](https://archive.softwareheritage.org/badge/origin/https://github.com/ARUP-CAS/aiscr-webamcr/)](https://archive.softwareheritage.org/browse/origin/?origin_url=https://github.com/ARUP-CAS/aiscr-webamcr) [![OpenSSF Best Practices](https://www.bestpractices.dev/projects/8934/badge)](https://www.bestpractices.dev/projects/8934) [![Documentation Status](https://readthedocs.org/projects/aiscr-webamcr/badge/?version=latest)](https://aiscr-webamcr.readthedocs.io/cs/latest/?badge=latest) [![Licence: GPL v3](https://img.shields.io/badge/License-GPLv3-blue.svg)](https://www.gnu.org/licenses/gpl-3.0)

The Archaeological Map of the Czech Republic (AMCR) is an information system for collecting, managing, and presenting data on archaeological fieldwork and for supporting related workflows. It is operated by the Institute of Archaeology of the Czech Academy of Sciences in Prague and Brno.

## Links

- **Documentation:** <https://aiscr-webamcr.readthedocs.io/>

## Development environment

### Prerequisites

- Python 3.11+
- Docker and Docker Compose
- Node.js 18+ and npm (for SCSS / JS build)

---

### Running locally

```bash
# 1. Create a virtualenv and install dependencies
python -m venv .venv
.venv/Scripts/activate   # Windows
# source .venv/bin/activate  # Linux / macOS
pip install -r requirements.txt

# 2. Start infrastructure services
docker compose -f docker-compose-dev-local-db-all-containers.yml up -d

# 3. Start Django
python manage.py migrate
python manage.py runserver

# 4. Start Celery worker (in a separate terminal)
celery -A webclient worker -l info
```

Detailed installation guide:  
<https://aiscr-webamcr.readthedocs.io/>

### Environment configuration

Sensitive values (passwords, API keys, `SECRET_KEY`) are **never committed**.

Use:

- environment variables
- `.env` file (see `.gitignore`)

> **Note on `cert/`:**  
> The directory contains self-signed certificates used only for local development.  
> They are intentionally committed and do not represent a security risk in production.

## AI agents

The repository includes configuration for AI coding agents (OpenAI Codex, Claude Code, and others):

- `AGENTS.md` — rules, conventions, and instructions for agents
- `.agents/` — ongoing audits, analyses, and backlog of findings

Branches generated by agents:

```markdown
agents/{agent_name}/<topic>
```

are branched from `test` and merged back into `test` **only after human review**.
