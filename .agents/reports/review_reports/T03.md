## 2. Identifikované problémy

### 2.1 Použití eval() na hodnotách z databáze (BUG-001)

**Soubory:**
- `webclient/projekt/models.py:663` — `Projekt.set_permanent_ident_cely()`
- `webclient/arch_z/models.py:331` — `ArcheologickyZaznam.set_lokalita_permanent_ident_cely()`
- `webclient/arch_z/models.py:889` — `get_akce_ident()`

Ve všech třech případech se používá `eval(i)` k převodu textového čísla (části `ident_cely`) na integer. Správnou náhradou je `int(i)`. Ačkoliv jsou hodnoty `ident_cely` generovány interně a jejich formát je kontrolovaný, `eval()` je špatná praxe a představuje zbytečné bezpečnostní riziko.

```python
# Aktuální (špatně):
idents = [eval(i) for i in idents]

# Správně:
idents = [int(i) for i in idents]
```

#### NP1-008, NP1-012 — Extra SELECT v save() metodách

Jak `ArcheologickyZaznam.save()` (arch_z/models.py:531), tak `SamostatnyNalez.save()` (pas/models.py:182) dělají při každém uložení záznamu extra `SELECT` pro zjištění změny pristupnost:

```python
def save(self, *args, **kwargs):
    if self.pk is not None:
        previous = ArcheologickyZaznam.objects.get(pk=self.pk)  # extra SELECT vždy
```

Toto lze řešit ukládáním počáteční hodnoty v `__init__` (jako u `initial_stav`).

### 2.4 Import cached_property z distlib (BUG-003)

`uzivatel/models.py:28`:
```python
from distlib.util import cached_property
```

`distlib` je balíčkovací utilita, nikoliv Django komponenta. `cached_property` by mělo být importováno z `functools` (Python 3.8+) nebo `django.utils.functional`. Tento import je funkčně správný, ale nespolehlivý a matoucí — `distlib` není zaručeně dostupný v produkčním prostředí jako přímá závislost.

### 2.5 Deprecated .extra() v arch_z/filters.py

Řádky 773–775 používají `.extra(where=["ST_Z(geom) >= %s"])`. Metoda `.extra()` je deprecated od Django 4.0. Nahrazením by bylo `RawSQL()` nebo přidání GIS-specifické funkce.

## 3. Návrhy zlepšení

### 3.1 Nahradit eval() za int()
Jednoduché a bezpečné nahrazení — tři místa v kódu. Priorita: vysoká.

### 3.2 Prefetch v check_pred_* metodách
Přidat `prefetch_related` tam, kde jsou check metody volány na querysetech:
```python
# Příklad pro Projekt.check_pred_uzavrenim()
projekt = Projekt.objects.prefetch_related(
    'akce_set__archeologicky_zaznam__dokumentacni_jednotky_akce__komponenty__komponenty',
    'akce_set__archeologicky_zaznam__casti_dokumentu__dokument'
).get(pk=pk)
```

### 3.3 Cached initial_pristupnost v __init__
Stejný vzor jako pro `initial_stav` — uložit počáteční hodnotu pristupnost v `__init__`:
```python
def __init__(self, *args, **kwargs):
    super().__init__(*args, **kwargs)
    self.initial_stav = self.stav
    self._initial_pristupnost = self.pristupnost  # nové
```
Tím odpadá extra SELECT v save().

### 3.4 Bulk_update místo individuálních save() v User.save()
```python
# Místo:
for spoluprace in spoluprace_query:
    spoluprace.stav = SPOLUPRACE_NEAKTIVNI
    spoluprace.save()

# Použít:
spoluprace_query.update(stav=SPOLUPRACE_NEAKTIVNI)
```

### 3.5 Přidat indexy na pas.SamostatnyNalez
Přidat `db_index=True` nebo migrací přidat indexy na nejčastěji filtrovaná pole: `stav`, `projekt`, `katastr`, `pristupnost`.

### 3.6 Opravit import cached_property
```python
# Místo:
from distlib.util import cached_property

# Použít:
from functools import cached_property
```

### 3.7 Nahradit .extra() za RawSQL nebo ORM alternativu
Pokud PostGIS Django binding nepodporuje Z-souřadnici nativně, použít:
```python
from django.db.models.expressions import RawSQL
queryset = queryset.filter(geom__z__gte=value)
# nebo:
queryset = queryset.annotate(z_coord=RawSQL("ST_Z(geom)", [])).filter(z_coord__gte=value)
```
