## 1. Summary of findings

The repository contains a total of **152 migrations** across 11 applications — the largest are uzivatel (31), core (26), arch_z (20), and dokument (19). The models are generally well indexed, except for `pas.SamostatnyNalez` and `uzivatel.NotificationsLog`. **12 N+1 candidates** were identified, mostly in `check_pred_*` and model `save()` methods, **3 occurrences** of the dangerous `eval()` pattern on values from the database, and **1 suspicious import** from a third party (distlib). Spatial (PostGIS) queries are implemented via `raw()` — a legitimate use, but `.extra()` in `arch_z/filters.py` is deprecated.

## 2. Identified problems

### 2.1 Use of eval() on values from the database (BUG-001)

**Files:**
- `webclient/projekt/models.py:663` — `Projekt.set_permanent_ident_cely()`
- `webclient/arch_z/models.py:331` — `ArcheologickyZaznam.set_lokalita_permanent_ident_cely()`
- `webclient/arch_z/models.py:889` — `get_akce_ident()`

In all three cases, `eval(i)` is used to convert a textual number (part of `ident_cely`) to an integer. The correct replacement is `int(i)`. Although the `ident_cely` values are generated internally and their format is controlled, `eval()` is bad practice and represents an unnecessary security risk.

```python
# Current (wrong):
idents = [eval(i) for i in idents]

# Correct:
idents = [int(i) for i in idents]
```

### 2.2 N+1 queries in the check_pred_* methods

#### NP1-002 — Projekt.check_pred_uzavrenim() — High severity

`projekt/models.py` calls `self.akce_set.all()` twice and, inside the loop, runs `a.archeologicky_zaznam.check_pred_odeslanim()`, which itself makes queries:

```python
does_not_have_event = len(self.akce_set.all()) == 0  # 1st query
# ...
for a in self.akce_set.all():  # 2nd query
    akce_warnings = a.archeologicky_zaznam.check_pred_odeslanim()  # N queries
```

#### NP1-006 — ArcheologickyZaznam.check_pred_odeslanim() — High severity

Double call of `self.dokumentacni_jednotky_akce.all()` and nested iteration over the components:

```python
if len(self.dokumentacni_jednotky_akce.all()) == 0:  # 1st query
    ...
for dj in self.dokumentacni_jednotky_akce.all():  # 2nd query
    if not dj.negativni_jednotka and len(dj.komponenty.komponenty.all()) == 0:  # N queries
```

#### NP1-008, NP1-012 — Extra SELECT in the save() methods

Both `ArcheologickyZaznam.save()` (arch_z/models.py:531) and `SamostatnyNalez.save()` (pas/models.py:182) perform an extra `SELECT` on every record save to detect a change of pristupnost:

```python
def save(self, *args, **kwargs):
    if self.pk is not None:
        previous = ArcheologickyZaznam.objects.get(pk=self.pk)  # extra SELECT always
```

This can be solved by storing the initial value in `__init__` (as with `initial_stav`).

### 2.3 Missing indexes on pas.SamostatnyNalez

The `SamostatnyNalez` model has 11 FK fields without `db_index=True`:
`projekt`, `katastr`, `okolnosti`, `pristupnost`, `obdobi`, `druh_nalezu`, `specifikace`, `nalezce`, `predano_organizace`.

For a table with thousands of records, these indexes are key to filtering performance.

### 2.4 Import of cached_property from distlib (BUG-003)

`uzivatel/models.py:28`:
```python
from distlib.util import cached_property
```

`distlib` is a packaging utility, not a Django component. `cached_property` should be imported from `functools` (Python 3.8+) or `django.utils.functional`. This import is functionally correct, but unreliable and confusing — `distlib` is not guaranteed to be available in the production environment as a direct dependency.

### 2.5 Deprecated .extra() in arch_z/filters.py

Lines 773–775 use `.extra(where=["ST_Z(geom) >= %s"])`. The `.extra()` method has been deprecated since Django 4.0. A replacement would be `RawSQL()` or adding a GIS-specific function.

## 3. Improvement proposals

### 3.1 Replace eval() with int()
Simple and safe replacement — three places in the code. Priority: high.

### 3.2 Prefetch in the check_pred_* methods
Add `prefetch_related` where the check methods are called on querysets:
```python
# Example for Projekt.check_pred_uzavrenim()
projekt = Projekt.objects.prefetch_related(
    'akce_set__archeologicky_zaznam__dokumentacni_jednotky_akce__komponenty__komponenty',
    'akce_set__archeologicky_zaznam__casti_dokumentu__dokument'
).get(pk=pk)
```

### 3.3 Cached initial_pristupnost in __init__
Same pattern as for `initial_stav` — store the initial pristupnost value in `__init__`:
```python
def __init__(self, *args, **kwargs):
    super().__init__(*args, **kwargs)
    self.initial_stav = self.stav
    self._initial_pristupnost = self.pristupnost  # new
```
This eliminates the extra SELECT in save().

### 3.4 Bulk_update instead of individual save() in User.save()
```python
# Instead of:
for spoluprace in spoluprace_query:
    spoluprace.stav = SPOLUPRACE_NEAKTIVNI
    spoluprace.save()

# Use:
spoluprace_query.update(stav=SPOLUPRACE_NEAKTIVNI)
```

### 3.5 Add indexes to pas.SamostatnyNalez
Add `db_index=True` or add indexes via a migration to the most frequently filtered fields: `stav`, `projekt`, `katastr`, `pristupnost`.

### 3.6 Fix the cached_property import
```python
# Instead of:
from distlib.util import cached_property

# Use:
from functools import cached_property
```

### 3.7 Replace .extra() with RawSQL or an ORM alternative
If the PostGIS Django binding does not support the Z coordinate natively, use:
```python
from django.db.models.expressions import RawSQL
queryset = queryset.filter(geom__z__gte=value)
# or:
queryset = queryset.annotate(z_coord=RawSQL("ST_Z(geom)", [])).filter(z_coord__gte=value)
```
