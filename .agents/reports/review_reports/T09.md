# T09 — Analýza CI/CD (GitHub Actions, pre-commit, testy)

**Datum:** 2026-03-11  
**Dotčené soubory:** `.github/workflows/run-tests.yml`, `pre_commit.yml`, `deployment.yml`, `docker_scout.yml`, `pre_commit_update.yml`, `publish_images.yml`, `new-version.yml`, `.pre-commit-config.yaml`, `.flake8`, `.isort.cfg`, `docker-compose-test.yml`, `.agents/analysis/cicd_analysis.json`

## 1. Shrnutí zjištění

- CI/CD pipeline je poměrně komplexní a dobře rozdělená: zvlášť workflow pro pre-commit, image security (Docker Scout), release/tagging a publikaci Docker image, manuální deployment na DEV/TEST a spuštění Selenium testů na vzdáleném prostředí.
- Pre-commit infrastruktura je robustní: lokální hooky generují modulovou a Selenium dokumentaci, běhy v CI analyzují výstupy (docstring warnings, flake8 chyby) a v případě formátovacích změn automaticky otevírají PR s opravami.
- Linting nastavení ve `.flake8` a `.isort.cfg` je konzistentní s projektovou politikou (line length 120, vybrané ignore kódy, per-file ignores pro CI settings).
- Bezpečnostní scanning se zaměřuje na Docker image (Docker Scout + Trivy + SBOM + SARIF upload); chybí ale samostatný CodeQL workflow či Dependabot konfigurace pro Python závislosti.

## 2. Identifikované problémy

- **Chybějící Dependabot / statický code-scan workflow**
  - V repozitáři není `.github/dependabot.yml` ani samostatné CodeQL workflow; bezpečnostní nástroje jsou zaměřené hlavně na Docker image (Docker Scout, Trivy), nikoliv na samotný Python/Django kód a jeho závislosti.
- **Potenciálně těžší requirements v pre-commit workflowu**
  - Workflow `pre_commit.yml` vždy instaluje kompletní `webclient/requirements.txt` plus `pip-licenses` a `pre-commit`. Aktuální nastavení (cache + 15min timeout) je pravděpodobně dostačující, ale u větších změn nebo pomalejšího runneru by se mohlo přiblížit limitům.

## 3. Návrhy zlepšení

- **Doplnění bezpečnostního skenování kódu a závislostí**
  - Přidat `.github/dependabot.yml` pro sledování Python balíků a GitHub Actions verzí; minimalizuje riziko zastaralých závislostí s CVE i mimo Docker image.
  - Vytvořit lehké CodeQL workflow pro Python (spouštěné např. na push/PR do `dev`), aby doplnilo současný image-level scanning Trivy a Docker Scout.
- **Optimalizace pre-commit requirements (volitelné)**
  - Pokud se v budoucnu ukáže, že `pre_commit.yml` běhy jsou pomalé, zvážit oddělení minimální sady závislostí pro lint/format do samostatného requirements souboru a v CI instalovat právě ten místo plného `webclient/requirements.txt`.
