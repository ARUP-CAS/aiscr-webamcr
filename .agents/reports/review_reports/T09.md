# T09 — CI/CD analysis (GitHub Actions, pre-commit, tests)

**Date:** 2026-03-11  
**Affected files:** `.github/workflows/run-tests.yml`, `pre_commit.yml`, `deployment.yml`, `docker_scout.yml`, `pre_commit_update.yml`, `publish_images.yml`, `new-version.yml`, `.pre-commit-config.yaml`, `.flake8`, `.isort.cfg`, `docker-compose-test.yml`, `.agents/analysis/cicd_analysis.json`

## 1. Summary of findings

- The CI/CD pipeline is fairly complex and well split up: separate workflows for pre-commit, image security (Docker Scout), release/tagging and Docker image publishing, manual deployment to DEV/TEST, and running Selenium tests on a remote environment.
- The pre-commit infrastructure is robust: local hooks generate module and Selenium documentation, CI runs analyze the outputs (docstring warnings, flake8 errors), and in case of formatting changes they automatically open a PR with fixes.
- The linting configuration in `.flake8` and `.isort.cfg` is consistent with the project policy (line length 120, selected ignore codes, per-file ignores for CI settings).
- Security scanning focuses on the Docker image (Docker Scout + Trivy + SBOM + SARIF upload); however, a separate CodeQL workflow or Dependabot configuration for Python dependencies is missing.

## 2. Identified problems

- **Missing Dependabot / static code-scan workflow**
  - There is no `.github/dependabot.yml` or a separate CodeQL workflow in the repository; the security tools focus mainly on the Docker image (Docker Scout, Trivy), not on the Python/Django code itself and its dependencies.
- **Potentially heavier requirements in the pre-commit workflow**
  - The `pre_commit.yml` workflow always installs the complete `webclient/requirements.txt` plus `pip-licenses` and `pre-commit`. The current setup (cache + 15-min timeout) is probably sufficient, but with larger changes or a slower runner it could approach the limits.

## 3. Improvement proposals

- **Adding security scanning of code and dependencies**
  - Add `.github/dependabot.yml` to track Python packages and GitHub Actions versions; this minimizes the risk of outdated dependencies with CVEs beyond the Docker image.
  - Create a lightweight CodeQL workflow for Python (triggered e.g. on push/PR to `dev`) to complement the current image-level scanning by Trivy and Docker Scout.
- **Optimizing the pre-commit requirements (optional)**
  - If it turns out in the future that the `pre_commit.yml` runs are slow, consider separating the minimal set of dependencies for lint/format into a separate requirements file and installing exactly that in CI instead of the full `webclient/requirements.txt`.
