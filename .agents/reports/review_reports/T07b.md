## 1. Summary of findings — T07b

Analyzed 20 custom JS files (~3,846 lines). Updated the vendored exclusions in `review_config.yaml` (+2 files). Found 2 new bugs (BUG-016, BUG-017) and added 4 new backlog items (FRONT-03 through FRONT-06). In total, 7 high-severity issues, 10 medium, and 8 low.

### Key findings

- **BUG-016** (Medium): `form_fields_disabling.js:65` — the condition `element.type != 'select-multiple' || element.type != 'select-one'` is always true (OR of negations, De Morgan's law). The select/non-select branches are inverted.
- **BUG-017** (Medium): `coor_precision.js:31` — the function for JTSK rounding uses `global_fixed_precision.wgs84` instead of `.jtsk` for non-array inputs. Coordinates are rounded with the wrong precision.
- **Systemic pattern**: None of the map scripts implements an `xhr.onerror` handler — a network failure is silent (see FRONT-03).
- **Systemic pattern**: `JSON.parse(this.responseText)` is called without try/catch in 6 files (see FRONT-04).

## 4. Shared patterns across the map files

1. **Global variables:** Heavy reliance on global state (`global_map_can_edit`, `boundsLock`, `switchMap`, `poi_*`, `mcg`). The map files share the pattern of initializing globals from the Django templates.
2. **XHR without error handling:** None of the 5 map files implements `xhr.onerror`. A network failure is silent.
3. **JSON.parse without try/catch:** Present in 6 files — invalid JSON (e.g. a 500 HTML response) causes an uncontrolled failure.
4. **Leaflet API:** `clearLayers(arg)` called with an argument in 2 files — `clearLayers()` accepts no argument.
5. **Implicit globals:** Variables declared without `let`/`const`/`var` — they become implicit globals on `window`.

## 7. Prompt improvement proposals

- The T07 prompt should explicitly mention checking the correctness of the Leaflet API (clearLayers, bounds methods).
- Add a requirement to check conditional operators (`||` vs `&&`) in JS — De Morgan errors are common and hard to see.
- For vendored exclusions, the prompt should mention that files without a `/*!` header but with a `/**` header (like `datepicker-cs.js`) may be vendored and require a manual check.
