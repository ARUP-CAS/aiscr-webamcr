# T01 — Repository Map (Mapování struktury repozitáře)

**Datum:** 2026-03-10
**Dotčené soubory:** Kompletní struktura repozitáře (kořen, webclient/, scripts/, docs/, infrastruktura)

---

## 1. Shrnutí zjištění

Repozitář `aiscr-webamcr` je komplexní produkční Django webová aplikace pro Archeologickou mapu České republiky (AMČR). Skládá se z **23 Django aplikací**, **7 infrastrukturních komponent** a rozsáhlé dokumentace (~261 RST souborů).

**Klíčové statistiky:**
- 23 Django aplikací v `webclient/`
- ~190 migrací celkem (největší počet: `uzivatel` s 31 migracemi)
- 77 Python souborů v `core/` — výrazně největší aplikace
- Část aplikací má vlastní testy; několik aplikací (`cron`, `fedora_management`, `healthcheck`, `neidentakce`, `notifikace_projekty`, `services`, `vypis`, `xml_generator`) testy postrádá
- 7 GitHub Actions workflows
- 5 variant docker-compose (prod, dev, proxy, test, git-deploy)
- 7 infrastrukturních komponent (Elasticsearch, Kibana, Logstash, Prometheus, Redis, Fedora, Nginx proxy)
- 18 deployment/maintenance skriptů v `scripts/` (deploy, runtime, databázové a testovací skripty)

**Jazykový přehled repozitáře:**
| Jazyk | Podíl |
|-------|-------|
| Python | 47,2 % |
| JavaScript | 36,8 % |
| HTML | 10,6 % |
| SCSS | 3,9 % |
| CSS | 0,7 % |

## 2. Identifikované problémy

### P01 — Aplikace bez testů
Šest aplikací nemá testovací pokrytí, přičemž dvě z nich obsahují Celery tasky nebo notifikační logiku:

| Aplikace | Problém |
|----------|---------|
| `cron` | Obsahuje Celery tasky (tasks.py) — bez testů |
| `notifikace_projekty` | Obsahuje Celery tasky + views — bez testů |
| `neidentakce` | Doménová entita — bez testů |
| `fedora_management` | Správa Fedora repository — bez testů |
| `services` | Sdílené utility (mailer) — bez testů |
| `vypis` | Export dat — bez testů |
| `xml_generator` | Generátor XML exportů — bez testů |
| `healthcheck` | Health check endpoint — bez testů |

**Závažnost:** Střední (testování Celery tasků je kritické pro spolehlivost)

### P02 — Koncentrace logiky v `core`
Aplikace `core` obsahuje 77 Python souborů a 26 migrací. Je zodpovědná za autentizaci, konektory, middleware, coordTransform, management commands a sdílené modely. Tato vysoká koncentrace zodpovědností je architektonický dluh.

**Závažnost:** Střední (refaktoring by zvýšil přehlednost, ale není akutní)

### P03 — Nekonzistentní pojmenování skriptu
Soubor `services/notfication_settings.py` obsahuje překlep (`notfication` místo `notification`).

**Závažnost:** Nízká

### P04 — Konfigurační soubory `.example` bez dokumentace procesu kopírování
Infrastrukturní komponenty používají `.example` soubory (elasticsearch.example.yml, kibana.example.yml, logstash.example.conf, prometheus.example.yml). Postup jejich aktivace by měl být zdokumentován.

**Závažnost:** Nízká

---

## 3. Návrhy zlepšení

1. **Přidat testy do `cron` a `notifikace_projekty`** — tyto aplikace obsahují Celery tasky, jejichž selhání je těžko detekovatelné bez testů.
2. **Zvážit postupné rozdělení `core`** — např. vyčlenit `coordTransform`, `ident_cely` a `repository_connector` do samostatných utility modulů.
3. **Opravit překlep** v `services/notfication_settings.py`.
4. **Zdokumentovat aktivaci `.example` konfiguračních souborů** v deployment dokumentaci.
